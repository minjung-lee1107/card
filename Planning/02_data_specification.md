# 데이터 명세서

## 1. 데이터 개요
본 데이터는 개인 지출 내역 분석을 위한 데이터로 CSV 또는 Excel 파일 형태로 입력된다. 
사용자의 소비 패턴 시각화 및 AI기반 인사이트 도출을 목적으로 사용된다. 

## 2. 데이터 출처
본 데이터는 개인 카드 사용 내역을 참고하여 설계되었다.
소비분석과 직접적인 연관이 없는 컬럼은 제외하고 분석에 필요한 핵심 컬럼만 사용한다. 

## 3. 데이터 스키마
| 컬럼명 | 타입 | 필수 | 설명 | 예시 |
|------|------|------|------|------|
| date | DATE | O | 거래날짜 | 2026-02-03 |
| amount | INTEGER | O | 금액(원) | 15000 |
| category | STRING | O | 상위 지출 카테고리 | 식비 |
| description | STRING | O | 거래 내역 설명 | 스타벅스 |
| sub_category | STRING |  | 세부 지출 분류 | 카페 |
| payment_method | STRING |  | 결제 수단 | 카드 |
| is_fixed | BOOLEAN |  | 고정비 여부 | FALSE |
| installment_type | STRING |  | 결제 방식 (일시불/할부) | 일시불 |
| installment_months | INTEGER |  | 할부 개월 수 | 3 |

### installment 관련 컬럼
할부 정보는 고액 지출 분석 시 활용 가능하도록 선택적으로 설계하였다.
본 프로젝트에서는 월별 지출 분석을 중심으로 하여 기본 분석에서는 할부 개념을 단순화하여 사용한다.

단, installment_months는 installment_type이 '할부'인 경우에만 값이 존재한다.

### is_fixed
구독, 월세, 관리비, 통신비, 보험료, 대출상환은 고정비로 분류한다.
그 외 항목은 최근 3개월 이상 반복 발생하며 금액 변동 폭이 일정 수준 이하인 경우 고정비로 자동 판단한다.
다만 패턴 기반 고정비 판단은 주거/통신, 구독, 교육 등 고정비 성격을 가질 수 있는 카테고리에 한해 적용한다.
식비, 여가, 쇼핑과 같이 선택적 소비가 가능한 항목은 반복 패턴이 존재하더라도 고정비로 분류하지 않는다.

### payment_method
거래 발생 방식에 대한 참고 정보로 포함된다.
결제 수단에 따른 비교 분석이나 시각화는 수행하지 않는다.
모든 거래는 지출 금액 기준의 통합 분석 대상으로 처리한다.

## 4. 카테고리 분류 기준

| 카테고리 | 포함 항목 |
|---------|----------|
| 식비 | 식당, 카페, 배달, 장보기 |
| 교통비 | 대중교통, 택시, 주유, 주차 |
| 쇼핑 | 의류, 잡화, 온라인 쇼핑 |
| 주거/통신 | 월세, 관리비, 통신비, 인터넷 |
| 구독 | 넷플릭스, 유튜브, 멜론 등 |
| 의료/건강 | 병원, 약국, 헬스장 |
| 문화/여가 | 영화, 공연, 여행, 취미 |
| 교육 | 학원, 강의, 도서 |
| 금융/보험 | 보험료, 대출상환, 수수료 |
| 기타 | 위 카테고리에 해당하지 않는 항목 |

카테고리 분류는 다음 절차로 수행된다.
1. 거래 내역에 대해 키워드 기반 룰을 우선 적용
2. 분류되지 않은 항목에 대해 AI 모델을 활용한 자동 분류 수행

## 5. 데이터 활용 범위
본 데이터는 개인 소비 패턴 분석, 시각화, AI 기반 개선 인사이트 및 예산 추천을 위해 사용된다.
